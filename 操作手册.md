# 调度精灵 V6 操作手册

## 一、系统概述

调度精灵（DispatchHub）是上海港船舶代理调度Web应用，支持船期表管理、动态看板、申报确认、调度黑板、引航格式转换等功能。
- 线上地址：https://dylanwilde.github.io/diaodujingling-V2/
- 部署平台：GitHub Pages

## 三、上传船期表（核心操作）

### 3.1 操作步骤

1. 登录后，点击 **🚢 数据管理** TAB
2. 选择日期（默认为当天）
3. 点击 **📤 点击上传 .xls / .xlsx** 区域
4. 选择港务局下发的Excel船期表文件（支持同时选多个）
5. 系统自动解析并保存到浏览器数据库

### 3.2 自动发布

管理员（姜磊）上传后，系统会**自动发布到线上**，所有调度员刷新页面或等待10分钟自动刷新即可看到最新数据。首次使用需先手动发布一次以配置GitHub Token。

### 3.3 手动发布

管理员点击右上角 **📤 发布到线上** 按钮：
- 首次需输入 GitHub Personal Access Token
- Token 会安全存储在本地浏览器
- 发布后约1-2分钟线上生效
- 所有成员（含未登录访客）均可查看

### 3.4 注意事项

- 同一日期可多次上传，**新数据会覆盖旧数据**
- 上传文件格式：港务局标准船期表 .xls/.xlsx
- 系统自动识别：船名、航次、吃水、上下港、码头、英文名、ETA
- 标记行"此行不要删除"等会被自动过滤

## 四、动态看板（TAB 0）

### 4.1 功能说明

- 按码头分组展示所有船舶卡片
- 每张卡片显示：**船名、靠/开吃水、航次号、ETA（大号突出）、上下港、备注（单独行）**
- 统计面板：代理船舶数、24h预抵、48h预抵、码头数
- ETA ≤24h 显示红色标签，≤48h 显示橙色标签
- 点击卡片 → 弹窗查看船讯网实时AIS位置
- 支持按码头筛选、船名/航次搜索
- 每10分钟自动刷新数据

### 4.2 船讯网弹窗

- 显示船舶实时位置（经纬度）、航速、航向、吃水、目的地、ETA
- 支持跳转船讯网查看完整详情
- 需在浏览器控制台执行 `localStorage.setItem('shipxy_key', '你的密钥')` 配置船讯网API密钥

## 五、申报确认看板（TAB 1）

### 5.1 功能说明

- 按码头分组，展示所有待确认船舶
- ETA预警：🔴 <24h（红色） 🟠 <48h（橙色） 🟢 >48h（绿色）
- 点击船舶卡片 → 打开申报确认弹窗

### 5.2 确认7日海事

1. 点击船舶卡片
2. 勾选 **"已完成7日海事申报"**
3. 可填写备注（未完成原因等）
4. 点击 **💾 保存**
5. 已确认的卡片自动沉底，带 ✅ 标记和确认人姓名

## 六、港口海图（TAB 2）

静态展示上海港主要码头和锚地分布：
- **码头**：洋山深水港、洋山四期、外高桥1-5期、浦东、宝山、罗泾、军工路
- **锚地**：长江口1/2号、绿华山、洋山港、南槽、宝山锚地
- 点击码头/锚地标注可查看名称

## 七、调度黑板（TAB 3）⭐ 核心协作功能

### 7.1 功能说明

- 按日期组织的调度提醒/留言板
- **实时同步**：任何人发布的消息，其他登录用户8秒内自动看到
- 消息显示：发布人、角色标签、时间、内容
- 右上角显示当前同步状态

### 7.2 发布步骤

1. 登录后切换到 **📝 调度黑板** TAB
2. 选择日期（默认当天）
3. 在底部输入框输入内容
4. 按 Enter 或点击 **📨 发布**

### 7.3 消息管理

- **管理员**可删除任何消息（点击消息右上角 ✕）
- **发布者**可删除自己的消息
- 消息跟随数据一起发布到线上，所有成员同步接收

### 7.4 实时同步原理

- 管理员发布数据时，黑板消息一并打包
- 各成员浏览器每8秒自动拉取最新黑板消息
- 本地消息和线上消息自动合并去重

## 八、引航转换（TAB 4）

### 8.1 功能说明

将引航站原始格式数据转换为标准化格式：
- 拖轮名称标准化（港拖→港拖编号，沪救→救编号）
- 时间格式转换（MMDD/HH.MM → YYYY-MM-DD HH:MM）
- 泊位名称简化（洋山NO.X → 洋X）
- A/N标志处理

### 8.2 操作步骤

1. 粘贴引航站原始数据到左侧文本框
2. 点击 **✨ 转换**
3. 右侧显示转换结果
4. 点击 **📑 复制** 复制到剪贴板

## 九、数据管理（TAB 5）

### 9.1 船舶查询

- 输入船名或航次号，按 Enter 或点击 **🔍 查询**
- **搜索全部数据库**中所有日期的记录（不限于当前日期）
- 显示完整信息：船名、英文名、航次、码头、吃水、上下港、ETA、备注

### 9.2 导出Excel

- 选择日期后点击 **📥 导出Excel**
- 导出当前日期的船舶数据为 .xlsx 文件

### 9.3 备份与恢复

- **💾 备份全部**：下载整个数据库为JSON文件
- **📂 恢复备份**：上传之前备份的JSON文件恢复数据
- 建议定期备份，防止浏览器数据丢失

## 十、常见问题

### Q1: 上传后看板无数据？
- 确认已选择正确日期
- 确认Excel格式为港务局标准格式（船名列必须有"船名"表头）
- 刷新页面重试

### Q2: 发布失败？
- 确认已配置GitHub Token（管理员首次发布时会提示输入）
- Token需要 repo 权限
- 检查GitHub仓库 `DylanWilde/diaodujingling-V2` 是否存在

### Q3: 调度员无法编辑数据？
- 调度员只能编辑**当日**数据
- 如需编辑历史数据，请联系管理员（姜磊）

### Q4: 黑板消息不同步？
- 确认已登录（未登录无法使用黑板）
- 管理员发布后其他成员才能看到新消息
- 切换TAB后黑板重新加载

### Q5: 忘记密码？
- 联系管理员（姜磊）在账号管理中重置

## 十一、技术说明

### 数据存储
- **本地**：浏览器 IndexedDB（DDB_v4）
- **线上**：GitHub Pages 托管 data/ships.json
- 数据格式：`{ ships: [...], blackboard: [...], updated: timestamp }`

### 自动刷新机制
- 动态看板：每10分钟自动刷新
- 调度黑板：每8秒轮询线上数据
- 管理员上传后自动发布

### 浏览器兼容
- Chrome / Edge / Firefox 最新版
- 需要启用JavaScript和IndexedDB
